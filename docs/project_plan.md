# 설명형 보안 코파일럿 (SSafer)
당신의 코드가 세상에 안전하게 닿도록, SSafer

## 1. 프로젝트 개요

### 한 줄 정의
단일 EC2 + Docker 환경에서 배포 전 보안 설정을 자동 점검하고, 위험 원인과 해결 방법을 설명하며, 승인 기반으로 실제 수정까지 연결하는 보안 자동화 코파일럿

---

## 2. 기획 배경

- **비전공 개발자의 보안 진입장벽**
  - 서비스 개발에는 익숙하지만 인프라 보안은 놓치기 쉬움

- **기존 도구의 한계**
  - Trivy 등은 탐지는 가능하지만 결과가 어려워 이해하기 힘듦

- **파편화된 문제 해결**
  - AI는 맥락이 부족하고, 수동 수정 과정에서 오류 발생 가능

- **지속적인 관리의 부재**
  - 일회성 점검이 아닌, 상태 추적과 이력 관리가 필요

---

## 3. 타겟 사용자

- **바이브코더**
  - 복잡한 보안 지식 없이 안전하게 배포하고 싶은 사용자

- **SSAFY 취준생**
  - 보안 취약점과 대응 과정을 학습하고 싶은 사용자

---

## 4. 주요 기능 (Requirements)

| 구분 | 기능 |
|------|------|
| 인프라 관리 | 서버 등록 및 환경 정보 관리 (IP, Docker 등) |
| 점검 실행 | CLI 및 웹 기반 보안 점검 수행 |
| 점검 대상 | 포트, Docker 이미지, 설정 파일, Secret 노출 |
| 시각화 | 위험도 및 점검 결과 대시보드 |
| AI 설명 | 위험 원인, 영향, 해결 방법 자연어 설명 |
| 수정 및 검증 | 수정 제안 및 적용 후 재검증 |
| 운영 관리 | 감사 로그 및 알림 |

---

## 5. 시스템 아키텍처

### 사용자 환경
- CLI (`ssafer scan`) 실행
- docker-compose, env 등 분석
- 결과 JSON 생성 및 서버 전송

### 백엔드 (EC2)
- **Nginx + FastAPI:** API 처리
- **Trivy + Rule Engine:** 취약점 탐지 및 커스텀 룰 적용
- **LLM Adapter:** 결과 설명 및 수정안 생성
- **PostgreSQL:** 로그 및 이력 저장

### 프론트엔드
- React + TypeScript 기반 대시보드
- 수정안 비교(diff-view) UI

---

## 6. 차별점

- **설명 중심 보안 서비스**
  - 단순 탐지가 아닌 “왜 위험한지”까지 설명

- **통합 점검**
  - 포트, Docker, 설정, Secret을 한 번에 분석

- **전/후 비교**
  - 보안 상태 개선을 시각적으로 확인

- **이벤트 기반 분석**
  - 변화 감지 및 이상 징후 요약 제공

---

## 7. 오픈소스 전략

### 활용
- Trivy (취약점 탐지)
- react-diff-view (UI)
- FastAPI / Node.js (데이터 처리)

### 기여
- 보안 결과 자연어 해석 레이어 공개
- 취약점 설명 데이터셋 구축
- 자동 수정 템플릿 공유

---

## 8. 구현 로드맵

### MVP (핵심 기능)

1. EC2 + Docker 환경 점검
2. Trivy 기반 취약점 분석
3. 포트 노출 및 접근 범위 검사
4. docker-compose / YAML 분석
5. 설명형 결과 출력

---

### 확장 기능

6. 수정 코드 제안
7. 승인 기반 자동 수정
8. 이슈 이력 관리 및 재발 경고

---

### 고도화 기능

9. 전/후 비교 리포트
10. 로그 이벤트 요약

---

## 9. 기대 효과

- **보안 공백 최소화**
  - 초보 개발자도 안전한 배포 가능

- **학습 효율 향상**
  - 설명 기반으로 보안 개념 습득

- **생태계 기여**
  - 보안 지식 공유 및 수준 향상
