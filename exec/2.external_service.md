# 사용하는 외부 서비스 정보

1) Google OAuth
* 용도: Google 소셜 로그인 및 계정 연동
* 가입/설정 위치:
  * Google OAuth Console
  * OAuth 동의 화면 설정
  * OAuth 2.0 Web Client 생성
* 필요 정보/설정:
  * Google OAuth Client ID
  * Google OAuth Client Secret
  * 승인된 JavaScript 원본
    * 로컬: `http://localhost:5173`
    * 운영: `https://ssafer.co.kr`
  * 승인된 리디렉션 URI
    * 로컬: `http://localhost:5173/oauth/google/callback`
    * 운영: `https://ssafer.co.kr/oauth/google/callback`
* 사용 환경 변수:
  * Frontend: `VITE_GOOGLE_CLIENT_ID`, `VITE_OAUTH_REDIRECT_BASE_URL`
  * Backend: `GOOGLE_OAUTH_CLIENT_ID`, `GOOGLE_OAUTH_CLIENT_SECRET`, `GOOGLE_OAUTH_TOKEN_URI`, `GOOGLE_OAUTH_USER_INFO_URI`

2) GitHub OAuth
* 용도: GitHub 소셜 로그인 및 계정 연동
* 가입/설정 위치:
  * GitHub Developer settings
  * OAuth Apps
* 필요 정보/설정:
  * GitHub OAuth App Client ID
  * GitHub OAuth App Client Secret
  * Authorization callback URL
    * 로컬: `http://localhost:5173/oauth/github/callback`
    * 운영: `https://ssafer.co.kr/oauth/github/callback`
* 사용 환경 변수:
  * Frontend: `VITE_GITHUB_CLIENT_ID`, `VITE_OAUTH_REDIRECT_BASE_URL`
  * Backend: `GITHUB_OAUTH_CLIENT_ID`, `GITHUB_OAUTH_CLIENT_SECRET`, `GITHUB_OAUTH_TOKEN_URI`, `GITHUB_OAUTH_USER_INFO_URI`, `GITHUB_OAUTH_USER_EMAILS_URI`

3) AWS S3
* 용도:
  * CLI 및 웹 업로드 기반 스캔 원본 결과 저장
  * AI 분석 결과 JSON 저장
  * 분석 결과 다운로드용 presigned URL 발급
* 가입/설정 위치:
  * AWS Console
  * S3 Bucket
  * IAM User 또는 IAM Role
* 필요 정보/설정:
  * AWS Access Key ID
  * AWS Secret Access Key
  * AWS Region
  * Raw scan result bucket
  * Analysis result bucket
  * 브라우저 직접 업로드가 필요한 경우 S3 CORS 설정
* 사용 환경 변수:
  * Backend: `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `APP_SCAN_RAW_S3_BUCKET`, `APP_SCAN_RAW_S3_REGION`
  * AI/FastAPI/Worker: `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_REGION`, `APP_SCAN_RAW_S3_BUCKET`, `APP_ANALYSIS_RESULT_S3_BUCKET`

4) Resend Email API
* 용도:
  * 회원가입 이메일 인증 코드 발송
  * 비밀번호 재설정 코드 발송
* 가입/설정 위치:
  * Resend Dashboard
  * API Keys
  * Domains
* 필요 정보/설정:
  * Resend API Key
  * 발신 이메일 주소
  * 발신 도메인 인증
  * 이메일 인증 코드 TTL
  * 재전송 cooldown
  * 인증 완료 상태 유지 시간
* 사용 환경 변수:
  * Backend: `RESEND_API_KEY`, `EMAIL_FROM_ADDRESS`, `EMAIL_VERIFICATION_CODE_TTL_SECONDS`, `EMAIL_VERIFICATION_COOLDOWN_SECONDS`, `EMAIL_VERIFICATION_VERIFIED_TTL_SECONDS`

5) SSAFY GMS API
* 용도:
  * AI 분석 설명 생성
  * 보안 설정 수정안 생성
  * 분석 결과 검증
  * n8n MR 리뷰 자동화 워크플로우의 LLM 호출
* 가입/설정 위치:
  * SSAFY GMS API 사용 신청 또는 제공 계정
* 필요 정보/설정:
  * GMS API Key
  * GMS Base URL
  * 사용할 모델명
  * temperature
  * timeout
  * JSON 응답 포맷 사용 여부
* 사용 환경 변수:
  * AI/FastAPI: `LLM_PROVIDER`, `GMS_API_KEY`, `GMS_BASE_URL`, `GMS_MODEL`, `GMS_TEMPERATURE`, `GMS_TIMEOUT_SECONDS`, `GMS_FORCE_JSON_RESPONSE_FORMAT`, `LLM_EXPLAIN_MAX_TOKENS`, `LLM_FIX_MAX_TOKENS`, `LLM_VERIFY_MAX_TOKENS`

6) Anthropic Claude API
* 용도:
  * GMS 대신 사용할 수 있는 AI 분석용 외부 LLM provider
* 가입/설정 위치:
  * Anthropic Console
  * API Keys
* 필요 정보/설정:
  * Anthropic API Key
  * Claude 모델명
  * temperature
  * timeout
* 사용 환경 변수:
  * AI/FastAPI: `ANTHROPIC_API_KEY`, `ANTHROPIC_MODEL`, `ANTHROPIC_TEMPERATURE`, `ANTHROPIC_TIMEOUT_SECONDS`
* 비고:
  * 실제 운영 provider가 GMS인 경우 대체 provider 설정으로만 사용한다.

7) HasData Web Reference Search
* 용도:
  * AI 분석 중 취약점 및 보안 설정 관련 외부 참고 자료 검색
* 가입/설정 위치:
  * HasData API Dashboard
* 필요 정보/설정:
  * HasData API Key
  * 검색 사용 여부
  * 최대 검색 결과 수
  * timeout
* 사용 환경 변수:
  * AI/FastAPI: `HASDATA_API_KEY`, `HASDATA_ENABLED`, `HASDATA_MAX_RESULTS`, `HASDATA_TIMEOUT_SECONDS`

8) NVD API
* 용도:
  * CVE/NVD 취약점 참고 정보 조회
* 가입/설정 위치:
  * NVD API Key 발급 페이지
* 필요 정보/설정:
  * NVD API Key
  * Agent 기능 사용 여부
  * Agent 최대 반복 횟수
* 사용 환경 변수:
  * AI/FastAPI: `NVD_API_KEY`, `AGENT_ENABLED`, `AGENT_MAX_ITERATIONS`

9) LangSmith
* 용도:
  * AI chain 및 agent 실행 trace 확인
  * 분석 품질 디버깅 및 관찰
* 가입/설정 위치:
  * LangSmith
  * API Keys
  * Project
* 필요 정보/설정:
  * LangSmith API Key
  * endpoint
  * project name
  * tracing 사용 여부
* 사용 환경 변수:
  * AI/FastAPI: `LANGSMITH_TRACING`, `LANGSMITH_ENDPOINT`, `LANGSMITH_API_KEY`, `LANGSMITH_PROJECT`

10) Docker Hub
* 용도:
  * Jenkins 빌드 결과 Docker image 저장
  * 배포 서버에서 Spring, Nginx/Frontend, Engine, FastAPI/Worker image pull
* 가입/설정 위치:
  * Docker Hub
  * Access Token
  * Repository
* 필요 정보/설정:
  * Docker Hub namespace
  * Docker Hub username
  * Docker Hub access token 또는 password
  * image tag
* 사용 환경 변수:
  * Infra/Jenkins: `DOCKERHUB_NAMESPACE`, `IMAGE_TAG`, `SPRING_IMAGE`, `NGINX_IMAGE`, `ENGINE_IMAGE`, `FASTAPI_IMAGE`

11) GitLab / Jenkins
* 용도:
  * GitLab 소스 저장소 기반 CI/CD
  * Jenkins를 통한 Docker image build/push 및 EC2 배포
* 가입/설정 위치:
  * GitLab project
  * Jenkins credential
* 필요 정보/설정:
  * GitLab repository 접근 권한
  * Jenkins credential
  * Docker Hub credential
  * EC2 SSH key 및 접속 정보
  * 운영 `.env` 파일

12) Let's Encrypt / Certbot
* 용도:
  * 운영 도메인 HTTPS 인증서 발급 및 갱신
  * Nginx TLS 적용
* 가입/설정 위치:
  * 별도 계정 가입 없이 Certbot으로 인증서 발급
  * DNS가 운영 서버를 가리키도록 설정 필요
* 필요 정보/설정:
  * 운영 도메인
  * 80/443 포트 오픈
  * `/etc/letsencrypt` 인증서 디렉터리
  * `/var/www/certbot` HTTP-01 challenge 디렉터리
